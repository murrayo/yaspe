# Design: Add Hostname and Instance to Chart Titles

**Date:** 2026-07-06

## Goal

Append the system hostname and IRIS/Caché instance name (when available) to all chart titles generated by `yaspe.py`, so charts are self-identifying without needing to consult the overview file.

## Scope

`yaspe.py` charts only. `pretty_performance.py` titles (driven by `input.yml`) are excluded.

## Title Format

| Data available | Title format |
|---|---|
| Customer + hostname + instance | `"Glorefs - AcmeCorp (server01 / IRIS)"` |
| Customer + hostname, no instance | `"Glorefs - AcmeCorp (server01)"` |
| Customer only (no hostname in DB) | `"Glorefs - AcmeCorp"` (unchanged fallback) |

## Implementation

### New helper: `get_chart_title_base(connection)`

Added to `yaspe.py`. Queries the `overview` table for:
- `customer`
- `linux hostname` (falls back to `windows host name` if absent)
- `instance`

Returns the combined string per the table above. Edge cases:
- `instance` is `None` or `""` → omit it (Windows files, standalone `.mgst` files)
- Neither hostname key present → return `customer` unchanged
- `customer` is `None` → return `""` (safe, same as today)

### Callsite changes

Seven chart functions in `yaspe.py` each have:
```python
customer = execute_single_read_query(connection, "SELECT * FROM overview WHERE field = 'customer';")[2]
```

Replace each with:
```python
customer = get_chart_title_base(connection)
```

No other logic changes — titles are already built as `f"{metric} - {customer}"`.

**Affected functions:**
- `chart_vmstat`
- `chart_mgstat`
- `chart_perfmon`
- `chart_iostat`
- `chart_nfsiostat`
- `chart_aix_sar_d`
- `chart_free_memory`

## Files Changed

- `yaspe.py` — add `get_chart_title_base()`, update 7 callsites

## Out of Scope

- `pretty_performance.py` — uses `site_survey_input["Site Name"]` from `input.yml`, not touched
- `chart_templates.py` — no changes needed
- `chart_output.py` — no changes needed
